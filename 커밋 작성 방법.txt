[커밋 작성 방법]

1. 메세지 타입
1) feat : 새로운 기능을 추가하였을 때
2) fix : 버그를 수정하였을 때
3) docs : README 등 문서 내용을 변경하였을 때
4) style : 들여쓰기, 세미콜론 등을 변경하였을 때
5) refactor : 코드 리팩토링 했을 때 (* 기능의 변경은 없어야 한다.)
6) test : test코드의 작성 및 수정이 이루어졌을 때
7) chore : 외부 라이브러리 임포트 등의 작업을 완료했을 때

2. 기본 구조 : '메세지 타입 : 개발한 내용'

e.g. 스타일 변경한 경우
→ style : 변경한 내용

e.g. 코드 정리한 경우
→ refactor : 정리한 내용
